Part 1: Ray Generation and Scene Intersection

Walk through the ray generation and primitive intersection parts of the rendering pipeline.
Rays are generated for each coordinate point on a given image, to which the rays are shifted to be emitting from a camera located at the origin of a coordinate system for the camera space. Afterwards, the rays from the camera are converted into the proper world space that the primitives are located in. We also sample a select number of rays per pixel and average the color values to determine the color of a pixel in the image space, or what is displayed on the screen.

Explain the triangle intersection algorithm you implemented in your own words.
The triangle intersection algorithm that we used for this project is the Moller-Trumbore algorithm, which is an optimization of a typical ray-plane intersection algorithm. After computing the coefficients b1 and b2 with this algorithm, we use barycentric coordinates using the coefficients to determine if the point of intersection at time t (also calculated using the Moller-Trumbore algorithm) is within the triangle.

Part 2: Bounding Volume Hierarchy

Walk through your BVH construction algorithm. Explain the heuristic you chose for picking the splitting point.
Our BVH construction algorithm recursively splits the bounding box containing all of the primitives in a scene into smaller bounding boxes that are stored within one another in a tree-like structure. The location of the split is dependent on the average of the centroids for each of the primitives along each axis in order to achieve a balanced split between the primitives.

Two versions of the direct lighting function had to be implemented simulaneously in order to achieve the rendering views seen below. Part 4: Global Illumination

Walk through your implementation of the indirect lighting function.
The indirect lighting function revolves around a recursive implementation of the direct lighting function, where the rays that hit a point after one bounce are used again to sample more rays in the scene originating from that hit point. These rays take in some of the color from the hit point and continue bouncing until it reaches the maximum amount of bounces, to which it contributes to the estimation for the color of the pixel that it lands on. Base cases are used within this recursive function to regulate the amount of light that is returned from each bounce, such as if we choose to accumulate the light between bounces or not.

Part 5: Adaptive Sampling

Explain adaptive sampling. Walk through your implementation of the adaptive sampling.
Adaptive sampling revolves around the concept of convergence; certain pixels converge to their true values quicker than others, so the usage of a confidence interval can tell us about whether a pixel has likely converged or not. Our implementation of adaptive sampling uses a sum of illuminances collected from rays casted onto a single pixel, which determine the mean and variance of the value over the amount of samples collected. Then, per every interval of the parameter “samplesPerBatch”, we calculate the convergence and check if it falls within an acceptable range according to “maxTolerance”. If it does, we stop sampling for that pixel, since it is unlikely to change its value upon more sampling.
