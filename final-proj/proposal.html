Worst Birthday Party Simulator

Joshua Almario, Cesar Garcia Santana, Darren Ting, Ethan Ye

Link to webpage: cal-cs184-student.github.io/hw-webpages-veryspook/final-proj/proposal.html

Summary

Our project is a volumetric smoke simulation in a 3D environment created using the Unity game engine. We hope to accurately replicate scenarios by modeling smoke as an incompressible fluid.

Problem Description

Smoke particles can be created visually in Unity using the native particle systems within the engine, but there is currently no support for dynamic 3D smoke simulations that accurately depict the behavior of smoke in the real world, such as the cases where smoke fills a room or when a window is opened in a smoke-filled room. With our project, we hope to create a realistic depiction of smoke particles by treating it as an incompressible fluid. With this approach, we hope to accurately simulate smoke in a performance-friendly manner.

Plans

Questions that we hope to answer with our project are the following:
  • What are the components of smoke that control how it looks?
  • Is it viable to have real-time 3D smoke simulation?
  • What optimizations can improve performance and reduce system load when rendering fluids?
  • Can utilization of Unity’s built-in volumetric fog in tandem with fluid simulation create acceptable realistic looking smoke?
We would create a 3D demo within Unity that depict various situations in which smoke would be present. Scenes would include, as mentioned earlier, smoke progressively filling up a room or smoke being released out of an open window. The main demo would depict an indoor scene with a candle on a birthday cake producing smoke in a enclosed room. Our performance metrics would include FPS for performance, and divergence error for accuracy in modeling an incompressible fluid.

Stretch Goals

Our main stretch goal would be to make the scene interactive using Unity functionality. A moving camera that comes with player movement may cause complications in regards to rendering, which would make rendering more difficult, and would require our algorithms to be sufficiently efficient.
